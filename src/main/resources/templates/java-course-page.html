Java Theory

Introduction to Java

Java is a general-purpose programming language that is class-based, object-oriented, and designed to have as few implementation dependencies as possible.

Key Features of Java

  • Platform independence: Java programs can run on any operating system or device with a Java Virtual Machine (JVM) installed.
  • Object-oriented programming: Java supports encapsulation, inheritance, and polymorphism, making it suitable for building modular and reusable code.
  • Automatic memory management: Java uses a garbage collector to automatically deallocate memory occupied by objects that are no longer in use.
  • Robust and secure: Java includes features like exception handling, type checking, and security mechanisms to ensure reliable and secure code execution.
  • Rich standard library: Java provides a vast standard library that offers a wide range of classes and methods for common programming tasks.

Basic Syntax

Java programs are written in plain text files with a .java extension. The basic syntax of a Java program consists of classes, methods, variables, statements, and expressions.

Data Types in Java

Java supports various data types, including:

  • Primitive data types: byte, short, int, long, float, double, boolean, and char.
  • Reference data types: classes, interfaces, arrays, and enumerated types.

Variables in Java

Variables in Java are used to store and manipulate data. Here are some key points about variables:

  • Variables must be declared with a specific data type, such as int, double, or String.
  • Variable names should start with a letter, underscore (_), or dollar sign ($), and can include letters, digits, underscores, or dollar signs.
  • Java is case-sensitive, so "myVariable" and "myvariable" are considered different variables.
  • Variables can be assigned initial values when declared, or they can be assigned values later in the program.
  • Variables can be updated by assigning them new values using the assignment operator (=).
  • The final keyword can be used to declare constants, which hold values that cannot be modified.

Operators in Java

Java provides a variety of operators for performing operations on variables and values. Here are some commonly used operators:

  • Arithmetic operators: + (addition), - (subtraction), * (multiplication), / (division), % (modulus).
  • Assignment operators: = (simple assignment), += (addition assignment), -= (subtraction assignment), *= (multiplication assignment), /= (division assignment), %= (modulus assignment).
  • Comparison operators: == (equal to), != (not equal to), > (greater than), < (less than), >= (greater than or equal to), <= (less than or equal to).
  • Logical operators: && (logical AND), || (logical OR), ! (logical NOT).
  • Increment and decrement operators: ++ (increment by 1), -- (decrement by 1).
  • Bitwise operators: & (bitwise AND), | (bitwise OR), ^ (bitwise XOR), ~ (bitwise complement).
  • Ternary operator: ? : (conditional operator that assigns a value based on a condition).

Control Statements

Control statements in Java allow you to control the flow of execution in your program. Here are some commonly used control statements:

If-else Statement

The if-else statement allows you to execute a block of code based on a condition. It has the following syntax: 

if (condition) {
  // code to be executed if the condition is true
} else {
  // code to be executed if the condition is false
}

Example:

int number = 10;
if (number > 0) {
  System.out.println("The number is positive");
} else {
  System.out.println("The number is negative");
}

Switch Statement

The switch statement allows you to select one of many code blocks to be executed. It has the following syntax: 

switch (expression) {
  case value1:
    // code to be executed if the expression matches value1
    break;
  case value2:
    // code to be executed if the expression matches value2
    break;
  default:
    // code to be executed if the expression doesn't match any case
}

Example:

int day = 3;
String dayName;
switch (day) {
  case 1:
    dayName = "Monday";
    break;
  case 2:
    dayName = "Tuesday";
    break;
  case 3:
    dayName = "Wednesday";
    break;
  default:
    dayName = "Unknown";
}
System.out.println("The day is " + dayName);

Loops

Loops in Java allow you to repeatedly execute a block of code. Here are some commonly used loop statements:

For Loop

The for loop is used to iterate a block of code for a specified number of times. It has the following syntax: 

for (initialization; condition; increment/decrement) {
  // code to be executed in each iteration
}

Example:

for (int i = 1; i <= 5; i++) {
  System.out.println("Count: " + i);
}

While Loop

The while loop is used to repeatedly execute a block of code as long as a condition is true. It has the following syntax: 

while (condition) {
  // code to be executed as long as the condition is true
}

Example:

int i = 1;
while (i <= 5) {
  System.out.println("Count: " + i);
  i++;
}

Do-While Loop

The do-while loop is similar to the while loop, but it executes the code block at least once before checking the condition. It has the following syntax: 

do {
  // code to be executed
} while (condition);

Example:

int i = 1;
do {
  System.out.println("Count: " + i);
  i++;
} while (i <= 5);

Arrays

Arrays in Java are used to store multiple values of the same data type. Here's how you can work with arrays in Java:

Declaration and Initialization

To declare an array, you need to specify the data type of the elements it will store, followed by square brackets ([]), and the array name. You can then initialize the array with values using the assignment operator (=) and the array initializer syntax. 

data_type[] array_name = new data_type[] {value1, value2, value3, ...};

Example:

int[] numbers = new int[] {1, 2, 3, 4, 5};
String[] names = new String[] {"John", "Mary", "David"};

Accessing Array Elements

Array elements can be accessed using their index, which starts from 0. You can access an element by specifying the array name followed by the index enclosed in square brackets ([]). 

array_name[index]

Example:

int[] numbers = {1, 2, 3, 4, 5};
int firstNumber = numbers[0]; // Accessing the first element
String[] names = {"John", "Mary", "David"};
String firstName = names[0]; // Accessing the first element

Array Length

The length property of an array can be used to get the number of elements in the array. 

array_name.length

Example:

int[] numbers = {1, 2, 3, 4, 5};
int length = numbers.length; // Getting the length of the array

Collections in Java

Collections in Java provide a way to store and manipulate groups of objects. They offer more functionality and flexibility compared to arrays. Here are some commonly used collection interfaces and classes in Java:

List Interface

The List interface represents an ordered collection of elements where duplicates are allowed. Some implementations of the List interface include ArrayList, LinkedList, and Vector.

Set Interface

The Set interface represents a collection of unique elements where duplicates are not allowed. Some implementations of the Set interface include HashSet, TreeSet, and LinkedHashSet.

Map Interface

The Map interface represents a collection of key-value pairs, where each key is unique. Some implementations of the Map interface include HashMap, TreeMap, and LinkedHashMap.

Iterator Interface

The Iterator interface allows you to iterate over elements in a collection. It provides methods to check if there are more elements and to retrieve the next element.

Object-Oriented Programming (OOP) in Java

Java is an object-oriented programming language that incorporates several key features of OOP. These features enable the organization and structuring of code in a modular and reusable manner. The core OOP features in Java include:

Classes and Objects

Classes are the building blocks of Java programs. They define the structure, behavior, and properties of objects. Objects are instances of classes and represent individual entities in a program. They encapsulate data and behavior within a single unit.

Example:

class Car {
  // Properties
  String brand;
  String color;

  // Constructor
  Car(String brand, String color) {
    this.brand = brand;
    this.color = color;
  }

  // Method
  void startEngine() {
    System.out.println("Engine started");
  }
}

// Creating objects
Car myCar = new Car("Toyota", "Red");
myCar.startEngine();

Inheritance

Inheritance enables the creation of new classes (subclasses) based on existing classes (superclasses). It allows subclasses to inherit and extend the attributes and behaviors of their superclasses. Inheritance promotes code reuse and supports hierarchical relationships between classes.

Polymorphism

Polymorphism allows objects to be treated as instances of their own class or their superclass. It enables the flexibility to use objects of different types interchangeably. Polymorphism is achieved through method overriding and method overloading.

Encapsulation

Encapsulation is the bundling of data and related behaviors (methods) into a single unit called a class. It hides the internal implementation details of an object and provides access to its properties and methods through well-defined interfaces. Encapsulation ensures data integrity and facilitates code maintainability.

Abstraction

Abstraction focuses on representing essential features and behaviors of objects while hiding unnecessary details. It allows the creation of abstract classes and interfaces that define common characteristics and behaviors to be implemented by concrete classes. Abstraction helps achieve modularity and separation of concerns.

Interfaces

Interfaces define a contract specifying a set of methods that a class implementing the interface must implement. They provide a way to achieve multiple inheritances in Java and facilitate loose coupling between classes. Interfaces define a common set of behaviors that can be shared across different classes.

Composition

Composition is a form of object composition where objects are combined to create more complex objects. It involves creating a class that contains references to other objects as member variables. Composition allows building complex relationships and reusing existing functionality by creating objects that collaborate with each other.

Association

Association represents a relationship between two or more objects where each object maintains its separate identity. It can be a one-to-one, one-to-many, or many-to-many relationship. Objects can interact with each other through their associated relationships, enabling collaboration and interaction in a system.

Exception Handling in Java

Exception handling in Java allows for the graceful handling of runtime errors, known as exceptions. It helps prevent program crashes and provides a mechanism to recover from exceptional situations. Java provides a robust exception handling framework that includes the following key components:

Try-Catch Blocks

The try-catch block is used to enclose a section of code that might throw an exception. It allows you to catch and handle specific types of exceptions. If an exception occurs within the try block, the corresponding catch block is executed, allowing you to handle the exception gracefully.

Throwing Exceptions

In Java, you can explicitly throw an exception using the throw keyword. This allows you to create custom exceptions or propagate built-in exceptions. Throwing an exception interrupts the normal flow of execution and transfers control to the nearest catch block that can handle the exception.

Finally Block

The finally block is optional and is used to define code that should be executed regardless of whether an exception occurs or not. It is typically used to release resources, such as closing database connections or file streams, ensuring they are properly handled, even in the event of an exception.

Exception Types

Java provides a hierarchy of exception types, with the root class being java.lang.Exception. There are two main types of exceptions: checked exceptions and unchecked exceptions. Checked exceptions are checked at compile-time and must be explicitly handled or declared. Unchecked exceptions are not checked at compile-time and can be handled optionally.
